# Writeup

I built Glass Box, a trust-focused document pseudonymization experience for a user who is nervous about pasting sensitive information into an AI tool. The product accepts pasted text or uploaded files, detects personally identifiable information, replaces it with stable labels, and then makes the result inspectable. Each pseudonym label can be clicked to show the original value, entity type, confidence, and a plain-language reason for why it was flagged. The summary page turns the same data into a review workflow: category cards show what was found, each card opens a detailed list of flagged values, and the verification findings explain Clean, Risk, and Miss in words instead of symbols.

The main design choice was to treat trust as something earned through interrogation. A worried user does not only need a redacted document. They need to know why the tool touched one piece of text and left another piece alone. That is why I prioritized explainability surfaces over decorative polish: inline explanations, an audit trail, category details, a trust score qualifier, and a second verifier that looks for missed PII or residual re-identification risk. The two-agent structure is useful because the verifier has a different job than the detector. It audits the finished output as if someone were trying to re-identify the person from what remains.

I chose not to build a full collaborative editor, user accounts, permanent document storage, or a redaction drawing interface. Those features could be useful later, but they would distract from the core problem: helping Marcus decide whether this specific tool deserves trust. I also avoided storing documents in a database because persistence would create extra privacy risk for a demo about sensitive data. The system keeps the workflow focused on immediate processing, review, and explanation.

I also chose not to hide complexity behind a single "safe" badge. A score alone can feel comforting, but it is not enough for someone skeptical. Instead, the score is paired with evidence: the exact categories found, individual flagged values, confidence, reasons, verification findings, and an audit summary. The result is less like a magic redaction machine and more like a privacy review desk that shows its work.
